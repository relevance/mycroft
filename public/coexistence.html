Programming with values and references is liberating, but most of the JVM world doesn't work that way yet. In Java, and most other JVM languages, there is no way to tell if an object is a value or a reference. And in fact, most JVM objects are neither! As a result, reading values from an arbitrary Java object requires coordination through a different specific API for every object.

The best thing you can do in this situation is get out of APIs as quickly as possible, converting specific objects into generic data as soon as you possibly can.

Some examples:

  • XML data
  • JMX beans
  • Java reflection

(Note: Java reflection classes are known to be values. Even in such cases, you are usually better off converting them to generic data.)
